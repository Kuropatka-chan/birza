# Биржа DSM

Одностраничный сайт биржи с товарами, лотами на продажу и покупку, фильтрами и статистикой.

## Что реализовано

- Разбивка товаров по категориям и фильтр категорий на общем рынке.
- Переключение типа лотов (**продажа / покупка**) на общем рынке и в карточке товара.
- Привязка каждого лота к владельцу:
  - базовые лоты автоматически назначаются аккаунту `NotAHamster`;
  - пользовательские лоты в личном кабинете создаются от имени `Текущий пользователь`.
- Создание в личном кабинете как лотов на продажу, так и лотов на покупку.
- Сортировка лотов в карточке товара (цена, количество, владелец) с направлением сортировки.
- Статистика:
  - абсолютные значения по сделкам;
  - агрегация по периодам: **дни / недели / месяцы**;
  - фоновая история сделок за предыдущие 60 дней для каждого товара по двум потокам данных: продажи и покупки.

## Механика данных и статистики

### 1) Источник каталога товаров
- Файл `Биржа_с_ценами.txt` содержит список категорий и базовые цены товаров.
- Приложение читает только строки формата `Название — Цена ДСМ`.
- На основе базовой цены для каждого товара автоматически генерируются стартовые лоты покупки/продажи.

### 2) Фоновая статистика на 60 дней
- Файл `stats_sales_60_days.json` — ежедневная статистика **продаж** по каждому товару за 60 дней.
- Файл `stats_purchases_60_days.json` — ежедневная статистика **покупок** по каждому товару за 60 дней.
- Каждый объект в обоих файлах имеет структуру:

```json
{
  "date": "2026-01-05",
  "productName": "Древесина (сырьё, м³)",
  "avgPrice": 24.37,
  "quantity": 58
}
```

- Цены сгенерированы в диапазонах, близких к базовой цене товара, чтобы визуально не выбиваться из ценовой категории.
- При старте приложения эти файлы загружаются в `dealsHistory` как история сделок и используются в разделе статистики.

### 3) Пользовательские действия и влияние на статистику
- Любая подтверждённая сделка из карточки товара добавляется в `dealsHistory` поверх фоновых данных.
- Статистика в UI показывает сумму фоновых данных + новых сделок пользователя.

## Визуальный стиль

Интерфейс переведён в стиль **blue neon**:
- тёмная подложка с неоновыми синими/бирюзовыми акцентами;
- светящиеся границы интерактивных элементов;
- неоновая подсветка таблиц, кнопок и карточек.

Для дальнейшего развития проекта рекомендуется придерживаться той же палитры и визуальной логики (`styles.css`).

## Запуск

1. Запустите локальный сервер в корне проекта:

   ```bash
   python3 -m http.server 8000
   ```

2. Откройте в браузере `http://localhost:8000`.

## Файлы репозитория

- `index.html` — структура интерфейса и элементы управления.
- `styles.css` — визуальная тема blue neon и стили таблиц/форм/кнопок.
- `app.js` — загрузка данных, фильтрация/сортировка, сделки, создание лотов, статистика.
- `Биржа_с_ценами.txt` — исходные категории товаров и базовые цены.
- `stats_sales_60_days.json` — фоновая статистика продаж по дням (60 дней, все товары).
- `stats_purchases_60_days.json` — фоновая статистика покупок по дням (60 дней, все товары).
